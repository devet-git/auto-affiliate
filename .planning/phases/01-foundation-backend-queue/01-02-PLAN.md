# Plan 01-02: Celery and Redis Integration

## Objective
Tích hợp hàng đợi Celery với Redis broker để xử lý Background Tasks. Đáp ứng cấu trúc Domain-driven cho module sys_worker. Thỏa mãn CORE-02.

## Tasks

### 1. Add Queue Dependencies <automated>
- `<read_first>`: `server/requirements.txt`
- `<action>`: 
  - Append `celery` và `redis` vào file `server/requirements.txt`.
  - Mở `server/app/core/config.py`, thêm biến environment config: `REDIS_URL: str = "redis://localhost:6379/0"` vào class Settings. Khai báo tương tự vào `.env.example`.
- `<acceptance_criteria>`: `server/requirements.txt contains celery`

### 2. Initialize Celery Application <automated>
- `<read_first>`: `server/app/core/config.py`
- `<action>`: 
  - Tạo `server/app/core/celery_app.py`. 
  - Khởi tạo instace celery `celery_app = Celery("sys_worker", broker=settings.REDIS_URL, backend=settings.REDIS_URL)`. 
  - Viết cấu hình `task_routes = {"app.domains.sys_worker.tasks.*": "main-queue"}` cho `celery_app.conf`. 
- `<acceptance_criteria>`: `celery_app.py properly configures broker=settings.REDIS_URL`

### 3. Create First Celery Task <automated>
- `<read_first>`: `server/app/core/celery_app.py`
- `<action>`: 
  - Tạo file `server/app/domains/sys_worker/tasks.py`. 
  - Khai báo task mẫu `@celery_app.task(name="test_worker_task")` nhận tham số `msg: str`. Log thông điệp đó ra console bằng thư viện `logging` python chuẩn và return `{"status": "processed", "msg": msg}`.
- `<acceptance_criteria>`: `tasks.py utilizes @celery_app.task decorator`

### 4. Create API Trigger Endpoint <automated>
- `<read_first>`: `server/app/domains/sys_worker/tasks.py`
- `<action>`: 
  - Tạo `server/app/domains/sys_worker/router.py`. 
  - Define 1 endpoint nội bộ POST `/api/v1/worker/test`, lock lại bằng dependency `get_current_admin` từ `admin.dependencies`.
  - Body nhận `msg: str` từ Body JSON. Gọi hàm task `test_worker_task.delay(msg)`. 
  - Phản hồi HTTP JSON `{"task_id": "..."}`.
- `<acceptance_criteria>`: `router.py exposes POST endpoint calling test_worker_task.delay`

### 5. Register Admin Routers <automated>
- `<read_first>`: `server/main.py`, `server/app/domains/sys_worker/router.py`
- `<action>`: 
  - Cập nhật `server/main.py`. Import router từ `sys_worker` và `app.include_router()` nó vào application global.
- `<acceptance_criteria>`: `main.py includes sys_worker router`

## Verification
- Khởi động service redis (bằng Docker/Local).
- Chạy terminal 1: `celery -A app.core.celery_app worker -l info` trong thư mục server.
- Chạy terminal 2: `uvicorn app.main:app`
- Tại terminal 3, gửi 1 Curl POST đến `/api/v1/worker/test` với json body. Celery terminal phải hiện log đã nhận task và thông điệp.
