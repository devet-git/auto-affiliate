# Plan 04-02: Humanized Behavior & Warm-up Actions

<objective>
Implement "làm ấm" (warm-up) action sequences on the physical device to scroll through the news feed and occasionally pause, mimicking human behavior before executing the main automation intent.
</objective>

<tasks>

<task_group name="Warm-up Logic Implementation">
  <task>
    <description>Create a `warmup.py` service to abstract Appium swiping feed scrolling.</description>
    <read_first>
      - "app/domains/content_sourcing/services/appium_controller.py"
    </read_first>
    <action>
      Create "app/domains/content_sourcing/services/warmup.py".
      Implement `warmup_news_feed(driver, duration_sec: int = 30)`.
      Calculate end time based on `time.time() + duration_sec`. 
      Inside a `while` loop, use `driver.swipe(start_x=500, start_y=1500, end_x=500, end_y=500, duration=800)` to scroll the feed.
      Add `time.sleep(random.uniform(2, 5))` to mimic human reading and pauses.
    </action>
    <acceptance_criteria>
      - "app/domains/content_sourcing/services/warmup.py contains def warmup_news_feed("
      - "app/domains/content_sourcing/services/warmup.py contains driver.swipe("
    </acceptance_criteria>
  </task>

  <task>
    <description>Integrate warm-up sequence into `facebook_seeding.py` tasks</description>
    <read_first>
      - "app/domains/content_sourcing/services/facebook_seeding.py"
    </read_first>
    <action>
      In "facebook_seeding.py", import `warmup_news_feed` from `warmup`.
      In `comment_on_post` and `batch_comment`, call `warmup_news_feed(driver, duration_sec=30)` immediately after obtaining the driver, but before triggering the hard navigation (`driver.get(post_url)`).
    </action>
    <acceptance_criteria>

    </acceptance_criteria>
  </task>
</task_group>

</tasks>
