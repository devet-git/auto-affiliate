# Plan 04-03: Media Injection & Posting Automation

<objective>
Provide the capability to push external media (MP4) to the physical Appium device via `adb` and expose a task to upload it as a Reel/Post using the Facebook App's gallery picker.
</objective>

<tasks>

<task_group name="ADB Media Injector Component">
  <task>
    <description>Create an adb wrapper service to cleanly push media files to `/sdcard/DCIM/Camera` and trigger `MEDIA_SCANNER_SCAN_FILE`.</description>
    <read_first>
      - "app/domains/content_sourcing/services/appium_controller.py"
    </read_first>
    <action>
      Create "app/domains/content_sourcing/services/media_injector.py".
      Import `subprocess`.
      Implement `push_media_to_device(udid: str, local_file_path: str, remote_file_name: str = 'affiliate_vid.mp4') -> str`.
      Define `remote_path = f'/sdcard/DCIM/Camera/{remote_file_name}'`.
      Execute command `['adb', '-s', udid, 'push', local_file_path, remote_path]` via `subprocess.run(..., check=True)`.
      Execute command `['adb', '-s', udid, 'shell', 'am', 'broadcast', '-a', 'android.intent.action.MEDIA_SCANNER_SCAN_FILE', '-d', f'file://{remote_path}']` via `subprocess.run(..., check=True)`.
      Return `remote_path`.
    </action>
    <acceptance_criteria>
      - "app/domains/content_sourcing/services/media_injector.py contains def push_media_to_device("
      - "app/domains/content_sourcing/services/media_injector.py contains adb push"
      - "app/domains/content_sourcing/services/media_injector.py contains MEDIA_SCANNER_SCAN_FILE"
    </acceptance_criteria>
  </task>
</task_group>

<task_group name="Appium Media Upload Service">
  <task>
    <description>Add a new seeding function and Celery task to post an injected media file via FB Main.</description>
    <read_first>
      - "app/domains/sys_worker/seeding_tasks.py"
      - "app/domains/content_sourcing/services/facebook_seeding.py"
    </read_first>
    <action>
      In "facebook_seeding.py", implement `post_reel(udid: str, local_mp4_path: str, caption: str)`.
      In the implementation, call `push_media_to_device(udid, local_mp4_path)` (import from `media_injector`).
      Then call `get_driver(udid, app_type='main')` to start Facebook Main, which is responsible for video uploads. Return True for now (simulate success).
      In "seeding_tasks.py", expose a celery task named `exec_fb_post_reel` assigned to queue `appium_phone` with concurrency parameters similar to `exec_fb_comment` that accepts udid, local_mp4_path, and caption, and executes `post_reel`.
    </action>
    <acceptance_criteria>
      - "app/domains/content_sourcing/services/facebook_seeding.py contains def post_reel("
      - "app/domains/sys_worker/seeding_tasks.py contains exec_fb_post_reel("
    </acceptance_criteria>
  </task>
</task_group>

</tasks>

<must_haves>
- The system must trigger `android.intent.action.MEDIA_SCANNER_SCAN_FILE` immediately after adb push to refresh the Android Media store cache so standard app pickers bypass caching faults.
</must_haves>
