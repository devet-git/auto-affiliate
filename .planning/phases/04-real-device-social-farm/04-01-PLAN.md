# Plan 04-01: Appium Controller & Multi-App Context

<objective>
Refactor existing Appium single-app constraints. Create a dedicated `appium_controller.py` to manage Appium remote sessions dynamically, switching between Facebook Main and Facebook Lite (`com.facebook.katana` / `com.facebook.lite`). Update `facebook_seeding.py` and Celery tasks to support passing the `app_type` target context.
</objective>

    <read_first>
      - "app/domains/content_sourcing/services/facebook_seeding.py"
    </read_first>
    <action>
      Create "app/domains/content_sourcing/services/appium_controller.py".
      Implement `get_driver(udid: str, app_type: str = 'main')`.
      Require `from appium.options.android import UiAutomator2Options` and `from appium import webdriver`.
      If `app_type == 'lite'`, set `options.app_package = 'com.facebook.lite'` and `options.app_activity = 'com.facebook.lite.MainActivity'`.

      Return `webdriver.Remote('http://127.0.0.1:4723', options=options)`.
    </action>
