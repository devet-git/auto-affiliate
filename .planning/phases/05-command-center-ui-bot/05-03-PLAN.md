# Plan 05-03: Tích hợp Webhook Telegram Bot (Aiogram)

<objective>
Tạo Bot Telegram sử dụng framework `aiogram` v3 kết nối bằng Webhook chạy song song trên instance của FastAPI (TokenBasedRequestHandler). Bot nhận lệnh trực tiếp `/approve <id>` và đồng thời được trigger từ Celery Task để báo `Notification` Real-time khi Cào được Video.
</objective>

## Tasks

<task>
<read_first>
- server/app/main.py
- .env (cần cấu hình TELEGRAM_TOKEN, TELEGRAM_CHAT_ID, WEBHOOK_URL, WEBHOOK_SECRET)
</read_first>
<action>
1. Cài đặt thư viện `pip install aiogram`.
2. Tạo file `server/app/domains/notify/bot.py`. Viết 1 Async router với `Dispatcher` và `Bot(token=TELEGRAM_TOKEN)`.
3. Định nghĩa handler: `@dp.message(Command("approve"))`. Khi user gõ `/approve 123`, bot bóc ID, lookup db, và đổi Status sang Approved, rồi reply ("Đã duyệt ID 123!").
4. Cập nhật `main.py`, mount `TokenBasedRequestHandler` của `aiogram.webhook.aiohttp_server` (để dùng với FastAPI ta sẽ mapping Request Body Json hoặc dùng module `TokenBasedRequestHandler` / FastAPI mapping middleware) theo docs `aiogram` (feed_update manual cũng được) vào endpoint: `POST /api/webhook/telegram`.
</action>
<acceptance_criteria>
- File `server/app/domains/notify/bot.py` có chứa lệnh `dp.message.register(approve_cmd, Command("approve"))`.
- Cấu hình route FastAPI `/api/webhook/telegram` nhận input JSON, đẩy vào bot feed update.
- Biến môi trường .env.example cập nhật đủ key liên quan Telegram.
</acceptance_criteria>
</task>
