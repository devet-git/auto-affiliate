# Plan 05-01: Khởi tạo React Vite, Shadcn UI & Auth Flow

<objective>
Thiết lập bộ khung Frontend React Vite cho Dashboard Admin. Tích hợp thư viện UI `shadcn/ui` cơ bản và kết nối tới API Đăng nhập hiện có (`/auth/login`) sinh JWT token để bảo vệ toàn bộ các màn hình bên trong.
</objective>

## Tasks

<task>
<read_first>
- server/app/main.py (để xem cấu hình CORS backend)
- server/app/domains/auth/router.py (để xem cấu hình nhận token)
</read_first>
<action>
1. Tạo thư mục `server/ui`.
2. Khởi tạo Vite React TypeScript boilerplate vào trong thư mục đó (sử dụng npm create vite@latest).
3. Thêm TailwindCSS theo tài liệu Vite (cài đặt `tailwindcss @tailwindcss/vite`...).
4. Chạy `npx shadcn@latest init` với cấu hình: Style New York, Base color Slate, CSS variables true, RSC false. Path alias `@/components`.
5. Sinh component Button và Input: `npx shadcn@latest add button input form`.
6. Cập nhật `vite.config.ts` để proxy `/api` tới backend `http://localhost:8000`.
</action>
<acceptance_criteria>
- Thư mục `server/ui/src/components/ui` có chứa `button.tsx` và `input.tsx`.
- `server/ui/vite.config.ts` chứa rule proxy chuyển `^/api/.*` sang cổng `8000`.
</acceptance_criteria>
</task>

<task>
<read_first>
- server/ui/src/App.tsx
- server/app/domains/auth/router.py
</read_first>
<action>
1. Tạo file `server/ui/src/pages/Login.tsx`. Chứa một form đơn giản dùng `lucide-react` và `shadcn`.
2. Form yêu cầu nhập Username & Password, onSubmit gọi API `POST /api/auth/token` với form data `username` và `password`.
3. Nếu thành công, lưu `access_token` vào `localStorage` và chuyển hướng tới `/dashboard`.
4. Viết một `ProtectedRoute.tsx` bọc các router bên trong `/dashboard`.
</action>
<acceptance_criteria>
- `server/ui/src/pages/Login.tsx` gọi chính xác API backend với định dạng urlencoded form.
- Component `ProtectedRoute` xuất hiện trong sơ đồ Router của `App.tsx` hoặc `main.tsx`.
</acceptance_criteria>
</task>

## Verification
- Chạy `npm run dev` ở `server/ui`. Vào `localhost:5173/login`, điền mật khẩu admin đã thiết lập trong ENV và kiểm tra JWT token có được cấp không.
- Thử vào `localhost:5173/dashboard` khi chưa login sẽ bị đá về `/login`.
